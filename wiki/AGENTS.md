# Инструкция для ИИ-агентов (AGENTS.md)

Добро пожаловать в проект **Antigravity Bar**! Этот документ содержит правила, стандарты и соглашения, которые обязательны для выполнения любым искусственным интеллектом (агентом), работающим в данном репозитории.

---

## 🇷🇺 Языковые соглашения
1. **Комментарии в коде**: Все комментарии, документационные блоки (Docstrings) и сообщения об ошибках в выводе логов должны быть написаны на **русском языке**.
2. **Документация**: База знаний в директории `wiki/` ведется исключительно на **русском языке**.
3. **Ответы пользователю**: Все сообщения, отчеты и ответы пользователю формулируются на **русском языке** в вежливой и структурированной форме.

---

## 🛠 Соглашения по именованию (Naming Conventions)
В проекте приняты стандартные гайдлайны Apple для языка Swift:

- **Типы (Классы, Структуры, Протоколы, Перечисления)**: `PascalCase` (например, `AntigravityAPI`, `StatusBarUI`, `SystemEnvironment`).
- **Свойства, переменные, методы, аргументы**: `camelCase` (например, `remainingFraction`, `fetchQuotaInfo()`, `baseDir`).
- **Константы глобального уровня или перечисления**: `camelCase` (например, `.active`, `.disabled`).
- **Имена файлов Swift**: Совпадают с именем основного типа, объявленного в файле (например, `AppDelegate.swift`).
- **Вспомогательные файлы и скрипты**: `kebab-case` (например, `build-app.sh`) или `snake_case` (например, `archive_unused.py`).

## 🏗 Архитектура и структура папок
При внесении изменений придерживайтесь следующей структуры:

- `status-bar/Sources/AntigravityBar/` — основная бизнес-логика и интерфейс.
  - UI-компоненты (AppKit) должны размещаться или дополняться в `AppDelegate.swift` или `StatusBarUI.swift`.
  - Модули работы с системой (процессы, память, GPU, файловая система) должны находиться в `SystemStats.swift` или `TerminalHelper.swift`.
  - API запросы и работа с данными демона Google Antigravity — в `AntigravityAPI.swift`.
- `status-bar/Tests/AntigravityBarTests/` — тесты. Любые новые фичи, имеющие бизнес-логику (разбор ответов, форматирование времени, фильтрация данных), должны покрываться тестами в `AntigravityBarTests.swift`.

### Важное архитектурное правило (Изоляция ФС)
Для всех дисковых и системных операций (чтение каталогов `brain/`, `conversations/`, проверка существования файлов) класс `AntigravityAPI` использует протокол-обертку `SystemEnvironment`.
**Никогда не вызывайте `FileManager.default` напрямую в методах API**. Вместо этого используйте:
```swift
self.environment.fileAttributes(atPath: ...)
self.environment.removeItem(atPath: ...)
```
Это необходимо для того, чтобы юнит-тесты могли подменять реальную файловую систему на `MockSystemEnvironment` и выполняться в памяти без риска удаления пользовательских данных.

## 📝 Правила оформления коммитов (Commit Rules)
Используйте стандарт **Conventional Commits**:
- `feat`: Новый функционал (например, `feat: add support for adaptive polling intervals`).
- `fix`: Исправление ошибки (например, `fix: resolve crash when JSON daemon file is empty`).
- `docs`: Обновление документации (например, `docs: update setup guide with Xcode 15 steps`).
- `style`: Исправление стиля кодирования (пробелы, переносы строк, форматирование SwiftLint).
- `refactor`: Рефакторинг кода без изменения его внешнего поведения.
- `test`: Добавление или исправление тестов (например, `test: add unit test for cache size thresholds`).
- `chore`: Рутинные задачи сборки, конфигурации (обновление `.cursorrules` или `.gitignore`).

## 🧪 Валидация изменений (Verification Commands)
Перед завершением работы над задачей агент **обязан** выполнить следующие проверки:

1. **Компиляция проекта**:
   ```bash
   swift build --package-path status-bar
   ```
2. **Запуск тестов**:
   ```bash
   swift test --package-path status-bar
   ```
   *Все тесты должны завершаться без ошибок.*
3. **Проверка линтером**:
   ```bash
   swiftlint lint
   ```
   *Старайтесь минимизировать количество предупреждений. Не допускайте критических ошибок компиляции стиля (например, нарушения длины классов).*
