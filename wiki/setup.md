# Руководство по установке и настройке Antigravity Bar

Данное руководство содержит подробное описание системных требований, шагов сборки и установки, описание переменных окружения, а также инструкции по запуску тестов и вспомогательных скриптов диагностики.

---

## 📋 Требования к окружению (Prerequisites)

Для успешной компиляции и функционирования Antigravity Bar на вашем Mac требуются следующие программные компоненты:

1. **Операционная система:** macOS 13.0+ (Ventura, Sonoma, Sequoia или новее).
2. **Архитектура:** Apple Silicon (M1/M2/M3/M4) или Intel (x86_64).
3. **Среда разработки / Язык:** Swift 6.0+ (поставляется вместе с Xcode 16+ или Xcode Command Line Tools соответствующих версий).
4. **Инструмент сборки:** Swift Package Manager (SPM), интегрированный в инструментарий Swift.
5. **Внешние зависимости:**
   - Установленный и настроенный **Google Antigravity IDE** (или локально функционирующий демон `language_server`).
   - Системный пакетный менеджер **Homebrew** (`brew`) и клиент **Git** для работы встроенного анализатора технологического стека.

## 🛠 Пошаговая сборка и запуск

Приложение может быть запущено как в режиме разработки (прямо из исходного кода), так и собрано в виде готового macOS-бандла (`.app`).

### 1. Запуск в режиме разработки (Development)

Для быстрой компиляции и отладки без создания интерфейсного окна (запуск бинарного файла строки меню):

```bash
# Перейдите в папку с кодом приложения
cd status-bar

# Скомпилируйте и запустите проект
swift run AntigravityBar
```

Или с использованием стандартных команд SPM:

```bash
cd status-bar
swift build
.build/debug/AntigravityBar
```

### 2. Сборка готового приложения (`Antigravity Bar.app`)

Для повседневного использования рекомендуется собрать полноценное приложение статус-бара, которое будет поддерживать автозапуск при входе в систему.

Для автоматизации процесса сборки в папке `status-bar` находится скрипт `build-app.sh`. Чтобы выполнить сборку:

```bash
# 1. Перейдите в каталог status-bar
cd status-bar

# 2. Сделайте скрипт исполняемым (если необходимо)
chmod +x build-app.sh

# 3. Запустите сборку
./build-app.sh
```

**Что делает скрипт сборки:**
- Запускает сборку приложения в режиме оптимизации Release (`swift build -c release`).
- Создает полноценную структуру macOS бандла `Antigravity Bar.app` во временной папке.
- Встраивает метаданные (`Info.plist`), иконку приложения (`AppIcon.icns`) и ресурсы.
- Подписывает полученное приложение локальной ad-hoc подписью (`codesign`), что позволяет использовать службу управления автозапуском macOS (`SMAppService`).

После завершения сборки перенесите готовое приложение в системную папку приложений:

```bash
cp -r "Antigravity Bar.app" /Applications/
```

## ⚙️ Конфигурация и Переменные окружения (.env)

Приложение поддерживает работу с Telegram-мостом для удаленного управления IDE. Для активации этой функции необходимо настроить переменные окружения.

### Переменные для Telegram-моста

Вы можете создать файл `.env` в корне проекта или экспортировать переменные напрямую в вашу командную оболочку (например, в `~/.zshrc`):

| Переменная | Тип | Описание | Пример значения |
| :--- | :--- | :--- | :--- |
| `TG_BOT_TOKEN` | `String` | Токен вашего Telegram-бота, полученный от бота `@BotFather`. Используется для поллинга сообщений. | `123456789:ABCdefGhIJKlmNoPQRsT...` |
| `ALLOWED_USER_ID` | `Int` | Ваш личный числовой Telegram ID. Бот будет игнорировать сообщения от любых других пользователей. | `987654321` |

> [!IMPORTANT]
> Для обеспечения безопасности рекомендуется хранить секретный токен `TG_BOT_TOKEN` в системной связке ключей macOS Keychain, избегая жесткого кодирования секретов в файлах конфигурации.

## 📁 Структура папок и дисковые зависимости

Приложение автоматически сканирует домашний каталог пользователя для обнаружения активных сессий демона Google Antigravity. Оно поддерживает переключение между двумя базовыми директориями:
- `~/.gemini/antigravity/` — для классической версии Antigravity.
- `~/.gemini/antigravity-ide/` — для новой версии Antigravity IDE.

### Подкаталоги в рабочей директории демона:
- `brain/` — папка с логами и базой знаний агента. Именно её размер анализируется для предотвращения раздувания контекста (`Context Bloat`).
- `conversations/` — история диалогов с ИИ.
- `html_artifacts/` — временные HTML-файлы, сгенерированные агентом.
- `skills/` — установленные Markdown-инструкции для ИИ-агента.
- `global_workflows/` — глобальные сценарии автоматизации (воркфлоу).
- `registry.json` — локальный реестр репозиториев для менеджера пакетов навыков.

## 🧪 Тестирование и автоматическая проверка

Проект снабжен набором модульных тестов XCTest для верификации парсинга квот, подсчета размеров кэша и алгоритмов группировки моделей.

### 1. Запуск тестов
Для запуска тестов выполните команду:

```bash
swift test --package-path status-bar
```

> [!NOTE]
> В тестах используется абстракция `SystemEnvironment`. Благодаря этому все дисковые операции в тестах перехватываются мок-окружением `MockSystemEnvironment`, что полностью предотвращает случайное изменение или удаление реальных данных в домашнем каталоге пользователя.

### 2. Скрипт комплексной локальной проверки
В папку `wiki/scripts/` добавлен скрипт `check-project.sh`, который автоматизирует сборку, прогон тестов и проверку стиля с помощью SwiftLint.

Для запуска скрипта выполните из корня проекта:

```bash
./wiki/scripts/check-project.sh
```

### 3. Вспомогательные скрипты телеметрии
В папку `wiki/scripts/` перенесены служебные скрипты для точечной проверки сбора низкоуровневых системных метрик напрямую через API ядра macOS:
- Проверка загрузки CPU: `swift wiki/scripts/test_cpu.swift`
- Проверка загрузки GPU (IOKit/IOAccelerator): `swift wiki/scripts/test_gpu.swift`
- Проверка загрузки RAM (HOST_VM_INFO64): `swift wiki/scripts/test_ram.swift`
- Проверка импорта IOKit: `swift wiki/scripts/test_iokit.swift`
- Скрипт очистки и архивации неиспользуемых навыков: `python3 wiki/scripts/archive_unused.py`
