# Установка и настройка Antigravity Bar

В данном руководстве описаны системные требования, шаги для сборки и установки, параметры конфигурации окружения, а также инструкции по запуску тестов.

## Требования к окружению (Prerequisites)

Для успешной сборки и запуска приложения на вашем Mac необходимы следующие инструменты и компоненты:

- **Операционная система:** macOS 13.0+ (Ventura, Sonoma или Sequoia).
- **Среда разработки / Язык:** Swift 6.0+ (входит в состав актуальных версий Xcode или Command Line Tools).
- **Инструменты сборки:** Swift Package Manager (SPM).
- **Внешние зависимости:**
  - Установленный и запущенный **Google Antigravity IDE** (или соответствующий локальный демон `language_server`).
  - Установленные `brew` (Homebrew) и `git` для полноценной работы встроенного AI-анализатора стека и сканирования проектов.

## Сборка и установка

Приложение можно собрать и установить как в виде автономного `.app`-пакета для повседневного использования, так и запустить в режиме разработки.

### 1. Быстрый запуск в режиме разработки (Development)

Для компиляции и запуска консольного бинарного файла используйте стандартные команды SPM:

```bash
cd status-bar
swift build
.build/debug/AntigravityBar
```

### 2. Сборка готового приложения (`.app` Bundle)

Для сборки полноценного приложения строки меню macOS используется скрипт `build-app.sh`, расположенный в директории `status-bar`:

```bash
cd status-bar
chmod +x build-app.sh
./build-app.sh
```

Этот скрипт выполняет следующие действия:
- Компилирует проект в конфигурации Release (`swift build -c release`).
- Создает структуру macOS-бандла `Antigravity Stats.app` (включая иконку `AppIcon.icns` и файл конфигурации `Info.plist`).
- Применяет ad-hoc подпись к собранному приложению, что позволяет macOS корректно запускать его и использовать встроенные механизмы автозагрузки (`SMAppService`).

После успешного выполнения скрипта скопируйте приложение в системную папку программ:

```bash
cp -r "Antigravity Stats.app" /Applications/
```

## Переменные окружения и структура файлов

Antigravity Bar динамически считывает конфигурационные файлы и логи локального демона, а также хранит настройки AI-менеджера пакетов в домашней директории пользователя.

### Пути к рабочим каталогам

Приложение автоматически определяет версию запущенного демона и переключается между следующими базовыми каталогами в `~/.gemini/`:
- `~/.gemini/antigravity/` — для классической версии Antigravity.
- `~/.gemini/antigravity-ide/` — для новой версии Antigravity IDE.

### Структура подкаталогов

Внутри базовой директории используются следующие пути:
- `daemon/` — временные файлы обнаружения активного языкового сервера (содержат JSON с портами и токенами).
- `brain/` — локальная база знаний агента и история контекста.
- `conversations/` — история чатов и активных сессий.
- `html_artifacts/` — сгенерированные агентом HTML-артефакты.
- `skills/` — JIT-установленные файлы навыков (Markdown-инструкции для ИИ-агента).
- `global_workflows/` — глобальные сценарии автоматизации.
- `registry.json` — конфигурационный файл реестра репозиториев для скачивания навыков (поддерживается менеджером пакетов `InstallerView`).

## Тестирование

Проект содержит модульные тесты для проверки логики разбора квот, поиска демонов и очистки дискового пространства.

### Запуск тестов

Выполните команду в корне папки `status-bar`:

```bash
swift test
```

### Изоляция файловой системы в тестах

Для предотвращения случайного удаления реальных пользовательских файлов в тестах используется протокол `SystemEnvironment` и мок-окружение `MockSystemEnvironment`. В процессе тестов все файловые операции перехватываются и эмулируются в оперативной памяти.
