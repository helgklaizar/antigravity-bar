# Глоссарий предметной области (Glossary)

Данный глоссарий содержит определения ключевых терминов, сущностей и концепций, используемых в проекте **Antigravity Bar** и экосистеме **Google Antigravity IDE**.

---

## 🏛 Системные компоненты и процессы

### Языковой сервер (Daemon / Language Server)
Фоновый процесс `language_server_macos` (или `language_server`), запускаемый автоматически при старте Google Antigravity IDE. Он выполняет роль основного вычислительного ядра для интеллектуального анализа кода, автодополнения, синтаксического разбора и взаимодействия с облачными ИИ-моделями. Слушает локальные TCP-порты и обменивается данными по протоколу Connect.

### Компаньон строки меню (Status Bar Companion / AntigravityBar)
Нативное приложение macOS (`AntigravityBar`), функционирующее как вспомогательная утилита в системной строке меню (Menu Bar). Обеспечивает отображение телеметрии языкового сервера в реальном времени, управление квотами моделей, системный мониторинг хоста, а также предоставляет доступ к инструментам очистки кэша и установщику навыков.

### Telegram-мост (Telegram IDE Bridge)
Встроенный в статус-бар фоновый поллер Telegram Bot API, который прослушивает входящие сообщения от авторизованного пользователя (`ALLOWED_USER_ID`). Он позволяет удаленно управлять средой разработки на Mac (например, создавать новые чаты с конкретными задачами, открывать проекты через AppleScript и выполнять системный аудит).

### Менеджер пакетов навыков (AI Ecosystem Installer / InstallerView)
Интерактивный SwiftUI-интерфейс в составе статус-бара, предназначенный для JIT-установки (Just-In-Time) специализированных инструкций для ИИ-агента. Он сканирует установленный технологический стек в проектах пользователя и предлагает к установке соответствующие Markdown-файлы из подключенных репозиториев.

## 💾 Данные, Контекст и Память

### Контекстный кэш (Brain / Мозг)
Каталог `brain/` в рабочей папке демона (`~/.gemini/antigravity/` или `~/.gemini/antigravity-ide/`). Является локальным хранилищем логов работы ИИ-агента, промежуточных результатов рассуждений, индекса RAG (Retrieval-Augmented Generation) и статической памяти. Разрастание этого каталога напрямую влияет на производительность.

### Раздувание контекста (Context Bloat)
Эффект неконтролируемого накопления временных файлов, тяжелых логов диалогов (`conversations/`) и артефактов браузера (`browser_recordings/`) в рабочей папке демона. Это приводит к повышенному потреблению оперативной памяти языковым сервером и может вызывать микрофризы IDE при загрузке длинного контекста в контекстное окно ИИ.

### Очистка кэша (Clean Cache)
Служебная операция очистки временных файлов в рабочей директории демона. Включает удаление логов из `brain/`, истории диалогов из `conversations/` и сгенерированных HTML-страниц из `html_artifacts/`. При этом системный каталог знаний `knowledge/` намеренно сохраняется.

## 🧠 Навыки и Автоматизация ИИ

### Инструкции навыков (AI Skills / Системные промпты)
Специализированные файлы с расширением `.md`, расположенные в каталоге `skills/`. Они содержат инструкции на естественном языке, которые задают поведение ИИ-агента в определенных технологических сферах (например: `react-expert.md` — стандарты написания React-кода, `rust-memory-safety.md` — правила управления памятью в Rust).

### Глобальные сценарии автоматизации (Workflows / Воркфлоу)
Сценарии автоматизации, хранящиеся в папке `global_workflows/` в формате Markdown. Они описывают последовательности действий, которые ИИ-агент выполняет автономно (например: проверка типов, написание тестов, сборка, коммиты). Вызываются в чате с помощью синтаксиса `/имя_воркфлоу`.

### Холодное хранилище воркфлоу (Cold Storage / Workflows Archive)
Служебная директория `workflows_archive/`, предназначенная для хранения неиспользуемых сценариев автоматизации. Если анализатор выявляет сценарии, которые ни разу не запускались (0 вызовов по логам `overview.txt`), они перемещаются сюда для предотвращения засорения контекста.

### Реестр навыков (Registry JSON)
Конфигурационный файл `registry.json`, расположенный в корне рабочей директории демона. Он содержит структурированный список секций и адресов внешних репозиториев (например, на GitHub), содержащих навыки и воркфлоу. Используется менеджером пакетов для поиска и загрузки файлов.

## 📊 Телеметрия, Метрики и Безопасность

### Квоты моделей (Model Quotas)
Ограничения на использование облачных ИИ-моделей (Gemini 3.5, Gemini 3.1 Pro, Claude 3.5 Sonnet и др.), предоставляемые Google Antigravity. Статус-бар считывает их параметры: остаток лимита в процентах (`remainingFraction`) и точное время сброса лимита в формате ISO 8601 (`resetTime`), преобразуя их в наглядные элементы интерфейса.

### Адаптивный поллинг (Adaptive Polling)
Алгоритм оптимизации фоновой активности статус-бара. Он снижает частоту сетевых запросов к локальному порту демона и системных вызовов Darwin до одного раза в 10 секунд в фоновом режиме, но мгновенно выполняет обновление данных при открытии меню пользователем. Это экономит заряд батареи macOS.

### Обнаружение демона (Daemon Discovery)
Алгоритм автоматического поиска активных инстансов языкового сервера в операционной системе. Он сканирует список PIDs через системный API Darwin `proc_listpids`, отбирает процессы `language_server`, извлекает аргументы запуска с помощью `sysctl(KERN_PROCARGS2)` и определяет прослушиваемые порты через C-вызов `lsof`.

### Токен CSRF (X-Codeium-Csrf-Token)
Секретный токен сессии, генерируемый языковым сервером при старте. Он считывается статус-баром из аргументов процесса демона и подставляется в HTTP-заголовок каждого запроса к Connect API для защиты от атак типа CSRF (Cross-Site Request Forgery).

### Спарклайны ресурсов (Resource Sparklines)
Компактные графики-диаграммы без осей, отрисовываемые непосредственно в кнопке меню. Они визуализируют динамику использования ресурсов CPU, GPU и RAM на основе истории из последних 20 замеров телеметрии.

### Служба автозапуска (SMAppService)
Системный сервис macOS 13+, используемый статус-баром для регистрации приложения в автозагрузку операционной системы без необходимости создания устаревших Login Items или Launch Agents/Daemons вручную.
