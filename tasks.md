# Список задач и бэклог (Tasks & Backlog)

В данном файле ведется актуальный учет завершенных задач, текущего технического долга и планов по развитию проекта Antigravity Bar.

## 📋 Предстоящие задачи (Backlog)

### 🏗 Архитектура и Ядро
- [ ] **Оптимизация сбора процессов macOS (C-Interop):**
  - *Описание:* Полностью отказаться от вызова `lsof -p <PID>` для поиска открытого HTTP-порта демона, так как вызов внешней утилиты через `Process` создает лишнюю нагрузку.
  - *Решение:* Перейти к чтению таблиц сокетов и дескрипторов файлов процесса напрямую через системные вызовы Darwin / sysctl (сложно из-за C-interop структур, но повысит производительность).
- [ ] **Интеграция с локальным демоном по XPC / WebSockets:**
  - *Описание:* Согласно решению [ADR-0001](docs/adr/0001-swift-ui-decoupling.md), перевести коммуникацию на постоянное двустороннее соединение с фоновым сервисом для мгновенного получения логов событий и квот без polling-запросов.

### 🪄 Менеджер пакетов и Установка навыков (Installer)
- [ ] **Полноценное клонирование репозиториев в фоне:**
  - *Описание:* Сейчас кнопка `Install Selected` в установщике эмулирует процесс скачивания. Необходимо интегрировать вызовы `git clone` / `git pull` через `TerminalHelper` или libgit2 для реального скачивания файлов навыков.
- [ ] **Локальный кэш версий навыков:**
  - *Описание:* Добавить проверку обновлений для установленных `.md` файлов навыков с уведомлением пользователя, если в удаленном репозитории появилась новая версия.

### 📊 Телеметрия и UI
- [ ] **Настройка порогов кэша в настройках:**
  - *Описание:* Позволить пользователю переопределять пороги размера директории кэша (сейчас жестко зашито: 🟢 <100MB, 🟡 <300MB, 🟠 <500MB, 🔴 >500MB).
- [ ] **Экспорт системного отчета (Diagnostics):**
  - *Описание:* Добавить кнопку экспорта результатов сканирования `SystemAnalyzer` в файл `diagnostics.json` для упрощения отладки проблем пользователями.

## 🛠 Выполненные задачи (Changelog / Done)

### Спринт 2 (Текущий)
- [x] **Динамическое обнаружение портов демона (Dynamic Port Discovery):** Реализован поиск портов запущенного процесса `language_server` через быстрый точечный вызов `lsof` с флагами `-a -p <PID> -iTCP -sTCP:LISTEN -n -P`. Это решает проблему отсутствия связи с демоном, запущенным на случайном порту (когда `--https_server_port` равен `0`).
- [x] **Мгновенное обновление размера кэша/мозга:** Добавлен наблюдатель `didSet` на свойство `baseDir` в `AntigravityAPI`, чтобы размер кэша обновлялся в реальном времени при смене рабочей директории демона.
- [x] **Изоляция файловой системы (FileSystem Isolation):** Добавлен протокол `SystemEnvironment` и `MockSystemEnvironment` для безопасного модульного тестирования ФС.
- [x] **Адаптивный интервал опроса (Adaptive Polling):** Период опроса сокращается при открытом меню и увеличивается до стандартного в фоновом режиме, снижая энергопотребление.
- [x] **Безопасное обнаружение демона:** Заменен вызов утилиты `/bin/ps` на прямой опрос ядра Darwin через `proc_listpids` и `sysctl` (`KERN_PROCARGS2`), защищая приложение от зависания главного потока.
- [x] **Разбор квот через Decodable:** Жесткие ключи словарей переписаны на структуры Swift, соответствующие протоколу `Decodable`.
- [x] **Переименование бинарного файла:** Проект полностью переведен с рабочего имени `StellarBar` на `AntigravityBar` в файлах сборки (`Package.swift`, `build-app.sh`).
